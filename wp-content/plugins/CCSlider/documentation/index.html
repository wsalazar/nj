A) Introduction - top

CCSlider WP makes it easy to embed the CCSlider jQuery slideshow plugin, available at Codecanyon, in your Wordpress theme. CCSlider is unique from other jQuery based sliders because it provides 3d image effects. Overall there are 14 stunning 3d effects, and also 16 stylish 2d effects. CCSlider WP allows you to create multiple slideshows where each slideshow can have as many slides as you want. Although CCSlider WP allows you to create multiple slideshows you can only embed one slideshow per post/page.

Please note that the 3d effects will only work on browsers that support HTML5 Canvas. Currently this includes Firefox 3.0+, Safari 3.0+, Chrome 3.0+, Opera 9.5+ and IE 9+. Internet Explorer 6, 7 and 8 do not support Canvas. CCSlider allows you to set a 2d effect as a fallback effect for browsers that don't support Canvas. Hence visitors to your web page using old browsers will still see a nice slideshow.

B) Installing the plugin - top

To install the plugin follow the steps below:

  1. Log in to the Wordpress admin panel.
  2. Head over to Plugins section and click on the “Add New” button
  3. Then click “Upload” to bring up the plugin upload dialog. Select the plugin file and upload it. Alternatively you could also unpack the plugin zip file in your computer and then upload it to /wp-content/plugins via FTP.
  4. Next go to the Plugins main area and you will see CCSlider WP being listed as one of the available plugins.
  5. Click the “Activate” button next to it and you are done.

Please note that when CCSlider WP is installed then it creates a folder called ccslider-upload in /wp-content. So please ensure that this folder has correct permissions set. If you receive an error about the upload folder not being created then open /wp-content folder via FTP and set its permission to 777.

If you are upgrading from an earlier version then first deactivate the plugin. Then upload the new plugin files into /wp-content/plugins and overwrite the old files and after that activate the plugin. All your previously created slideshows will remain intact. Some of the new options that have been introduced in the current version will have default values as required by the plugin. This will ensure that your previously created slideshows keep on working. However it is best recommended that you check all your slideshows and see if they are working as you want. You can also tweak the values of the new options in the Settings page to your liking for a particular slideshow. If by any chance you find that the fields for the new option parameters in the slideshow settings page are empty after updating the plugin, then deactivate the plugin, logout of WP Dashboard, login again and then activate the plugin. After that the new option parameters will have the default values in all your saved slideshows.

C) How to Use - top

After activating the plugin you will see that a new menu item has been created on the left called CCSlider. This menu item has two sub menus - Slideshow Manager and Create New Slideshow. These names are self explanatory and whenever you want to create a new slideshow you will head over to the Create New Slideshow page, and all your existing slideshows will be listed in the Slideshow Manager page, from where you can edit or delete slideshows.

So first of all to create a new slideshow head over to the appropriate page and you will be presented with a list of options. They are described below:

Slideshow name

You must provide a name to the slideshow that you are going to create. This is a required field and you will receive an error if you leave it empty. Although different slideshows can have the same name but it is advisable that you provide unique names so that it is helpful for you to quickly differentiate between slideshows.

Slideshow description

You can write a description for the slideshow here. This field is optional.

Slide images, captions and links

Here you can upload images for use in the slider. Just click on the Upload a file button and a file dialog window will open. Select the image of your choice and upload it. In modern browsers you can also upload multiple images at a time. The image upload is handled through AJAX, and once the uploading is finished a list of boxes will appear where you can see a small version of the image uploaded and a textarea where you will be able to enter captions corresponding to that slide. As of version 2.0 a textbox is also presented where you can enter an url which will be used to open an external page when clicking on the corresponding slide image. CCSlider handles HTML captions so you can enter html content in the textarea. You can also drag the boxes containing the images and textareas to sort the order of the slides. When you hover over a box you will see a delete button appear on the left, on clicking which will delete the entire slide. Note that on clicking the delete button the slide image is deleted immediately. So if you reload the page after deleting a slide, without saving the settings first, then you will see a box with no image inside it.

Type of effect

Here you have the option to choose whether you want a 3d effect or a 2d effect. Depending on your choice further options will appear. If you choose 3d then a list of options for 3d effects will become visible, and if you choose 2d then its corresponding options will appear.

Options for 3d Slideshow

A list of options that are applicable when you choose a 3d effect. Their functions are described as follows:

  • Effect Name - the name of the 3d effect that you want. Choose one from the available list.
  • Image Width - the width of the rendered image. If your uploaded images are larger than this width then they will be automatically cropped to this width. The original images are however retained and you can change this parameter value again if you want.
  • Image Height - the height of the rendered image. Here also the uploaded images will be automatically cropped to this height and you can change this value anytime.
  • Wrapper Width - the width of the slideshow wrapper. Since 3d effects require space for animating, the wrapper width should be greater than the image width, by at least 100px, so that the 3d pieces don't get cut off while animating. You can check the slideshow preview and decide what width is actually needed.
  • Wrapper Height - the height of the slideshow wrapper and should be greater than image height because of the previous reason.
  • Inner Side Color - color of the inner side of the cube slices. Applicable only for "cube" 3d effects.
  • Transparent images - enable this if your slide images are transparent png images.
  • Enable Shadow - whether a shadow will be rendered or not. Applicable only for "cube" 3d effects.
  • Shadow Color - if you want a shadow then you can mention its color. Standard css color values apply.
  • Number of Slices - number of slices in which the image will be divided.
  • Number of Rows - number of rows in which the image will be divided. Applicable only for "grid" 3d effects
  • Number of Columns - number of columns in which the image will be divided. Applicable only for "grid" 3d effects
  • Delay - delay between the animation of each slice. Value is in ms.
  • Delay Direction - direction in which the delay will propagate. There are 4 available options.
  • Depth Offset - offset along the z-axis when animating the 3d effect.
  • Slice Gap - separation distance between the slices when animating the 3d effect.
  • Easing - an easing option of your choice. Generally easing values of the type "InOut" look much better.
  • FallBack - a 2d effect fallback for older browsers. Mention any effect from available list of 2d effects.
  • FallBack Speed - The time for the animation to be complete in case a 2d effect is activated for an older browser.

Options for 2d Slideshow

A list of options that are applicable when you choose a 2d effect. Their functions are described as follows:

  • Effect Name - the name of the 2d effect that you want. Choose one from the available list.
  • Slider Size - the width and height of the slider. The slide images will be automatically cropped to these dimensions. The original images will be retained and you can change the slider size anytime you want. The default value is 600 × 300 px.

General Options

A list of options is also presented that are applicable no matter what type of effect is chosen. Their functions are described below.

  • Animation speed - the time interval over which a particular effect will animate. Value should be in ms.
  • Start Slide - the slide number from which the slideshow will start. The numbering of slides is zero based, meaning that the first slide is 0, the second slide is 1 and so on.
  • Next/Prev buttons - this option lets you enable next/prev buttons in the slider.
  • Control Links - this option lets you enable control links to each slide, on clicking which a particular slide will come into view.
  • Thumbnail Control Links - this option lets you have thumbnails of the slide images as control links.
  • Thumbnail Size - if you chose to have thumbnails as control links then enter their dimensions here and the thumbnails will be automatically created. The default size is 100 × 50 px.
  • Autoplay - you can choose to enable autoplay of the slides.
  • Time interval for slides - here you can enter the length of time interval over which each slide will remain visible before the next slide appears.
  • Pause on hover - you can choose to pause the slideshow when the user hovers his mouse cursor over the slideshow.
  • Enable captions - you can choose to enable captions for your slideshow.
  • Caption animation - if captions are enabled then you can choose an animated effect for the captions from the available list.
  • Caption animation speed - the time period that the captions will take to animate.

Custom Transitions for each slide

As of version 2.0 you have the option to mention a custom transition for each slide. If you choose to have custom transitions then based on the type of effect you have chosen (3d or 2d) you will be provided with a list of options for each slide whose functions have been explained above. This list for the custom transitions for each slide is updated dynamically when you sort the list of uploaded slide images or delete a slide image, so that the order of the slides is maintained. You can also disable custom transitions for a slideshow anytime you want.

Whenever you want to create a new slideshow then the form on that page will be filled out with default option values, so that you don't have to fill out every single form field in order to make your slideshow work. You can just change the settings that you want to customize and save the slideshow settings. Needless to say, whenever you are going to edit an existing slideshow, the same page will open and all the form fields will be filled out with the settings with which you created the slideshow. Also you can preview your slideshow, with the settings that you had just entered, before saving it. The preview will also feature cropped slide images according to the dimensions you have specified, so that you can accurately visualize how your slideshow will look.

As already mentioned above, you have to remember one important fact concerning the width and height of the slider wrapper if you choose a 3d effect. The animations for 3d effects need some space, more than the width and height of the images. Whatever values you pass to the Image Width and Image Height plugin parameters described above, the width and height of the slider wrapper must be at least greater than that by about 100px. So for example if you set Image Width to 600px and Image Height to 300px then the width and height of the slider wrapper should be about 700px and 400px respectively.

If you do not set a higher width and height value then you will observe that while animating, parts of the images are being cut off. Therefore do not forget this. You will have to tweak the width and height values of the slider wrapper for a given value of the Image Width and Image Height parameters to find the optimal setting.

On the other hand if you choose a 2d effect then the plugin will automatically crop the images to the specified width and height of the slider. If you choose a 3d effect and a visitor to your site uses an older browser like IE so that the fallback effect is activated, then the slide images will be cropped to have the dimensions set in the Image Width and Image Height parameters.

If you choose a 3d effect and the fallback 2d effect is activated in an older browser like IE8, then a class called fallback is applied on the slider container. Also if you choose thumbnails for the control links then a class called controlThumbs is applied on the slider container. You can use these class names to apply special styling through css for these two cases.

Try not to set too large a width and height for the rendered images for the 3d effects, and don't set too many slices. The reason for this is that the 3d effects require more CPU processing power than the ordinary 2d effects. So too large images or too many slices will overload the CPU and result in choppy animations. Although modern browsers have fast javascript engines and also most modern computers have quite good processing power, but not all users have the latest browsers and hardware configuration. So an animation that appears smooth to one user may appear slow/jerky to another.

D) Embed slideshows - top

All the slideshows that you create will be visible in the Slideshow Manager page. When you go to that page you will see that each slideshow has a field in the table titled shortcode. In this field you will see the shortcode that is required for embedding a particular slideshow. Just copy the shortcode for a slider, that is displayed there and then head over to Posts section in Wordpress. Open any post or page where you want to have a slideshow, in the Wordpress editor, and then paste the shortcode at a place where you find suitable, and then save or publish that post or page. Then open that post/page in your browser and you will see the slider in action with all the settings that you created it with. If you want to directly write the embed code in a Wordpress template file then you will have to use the do_shortcode function. Using this function has the advantage that you can embed a slideshow in a part of the Wordpress template where content is not pulled using the Wordpress Loop. An example usage of this function to embed a slideshow is: 

    <?php echo do_shortcode('[ccslider id="1"]'); ?>
